# README: Входные Данные и Ограничения

## Описание

Этот проект предназначен для укладки коробок в грузовики с использованием различных алгоритмов. Он упрощает взаимодействие с системой через командную оболочку Spring Shell.

## Входные Данные

### Команды и Их Параметры

Команды доступны через интерфейс командной строки. Ниже приведены параметры, которые можно использовать с командой `stack` для укладки коробок:

- `--alreadyFilledTrucksFileName`: Имя файла JSON, содержащего уже заполненные грузовики (необязательный).
- `--boxesFileName`: Имя файла JSON, содержащего информацию о коробках (необязательный).
- `--typeAlgorithmName`: Имя алгоритма для укладки (по умолчанию — `burke`).
- `--truckSize`: Размер грузовиков, например, `10x10` (необязательный).
- `--limitNumberOfTrucks`: Максимальное количество грузовиков, которые может создать система (по умолчанию — `Integer.MAX_VALUE`).
- `--boxNames`: Имена коробок, разделенные запятыми (необязательный).

### Ограничения

1. **Файлы**: Необходимо предоставить верные имена файлов, которые существуют в системе и содержат корректные данные в формате JSON.
2. **Алгоритмы**: Поддерживаемые алгоритмы должны быть определены в системе (`burke`, `balanced`, `dummy` и другие).
3. **Размеры грузовиков**: Формат размера грузовиков должен соответствовать шаблону, например, `10x10`.
4. **Имена коробок**: Имена коробок не должны содержать специальных символов, за исключением запятых, которые разделяют имена.

## Примеры Запросов

Ниже приведены несколько готовых запросов, которые можно использовать для укладки коробок:

1. **Укладка с уже заполненными грузовиками и определенными коробками**:
   ```shell
   stack --alreadyFilledTrucksFileName trucks.json --boxesFileName boxes.json --typeAlgorithmName burke --truckSize 10x10 --limitNumberOfTrucks 10 --boxNames один,два
   ```

2. **Укладка с уже заполненными грузовиками**:
   ```shell
   stack --alreadyFilledTrucksFileName trucks.json --boxesFileName boxes.json --typeAlgorithmName burke --truckSize 4x10 --limitNumberOfTrucks 3
   ```

3. **Укладка для определенных коробок**:
   ```shell
   stack --alreadyFilledTrucksFileName trucks.json --boxesFileName boxes.json --typeAlgorithmName burke --truckSize 3x8 --boxNames один,два
   ```

4. **Укладка с использованием сбалансированного алгоритма**:
   ```shell
   stack --typeAlgorithmName balanced --limitNumberOfTrucks 3 --boxNames один,два,три,четыре,пять,шесть,семь,восемь,девять
   ```

5. **Укладка без указания коробок**:
   ```shell
   stack --limitNumberOfTrucks 3 --boxNames один,два,три,четыре,пять,шесть,семь,восемь,девять
   ```

6. **Укладка без указания других параметров**:
   ```shell
   stack --limitNumberOfTrucks 3
   ```

## Заключение

Используйте приведенные команды и параметры для эффективного взаимодействия с системой укладки коробок в грузовики. Убедитесь, что все входные данные корректны перед выполнением запроса.

## Дополнительная Информация о Параметрах

### Помощь по параметрам

Используйте следующие параметры для получения помощи при вводе команд:

- `ALREADY_FILLED_TRUCKS_FILE_NAME_HELPER`: Имя файла с уже загруженными грузовиками (.json).
- `BOXES_FILE_NAME_HELPER`: Имя файла с коробками (.json).
- `TYPE_ALGORITHM_NAME_HELPER`: Имя алгоритма (burke, balanced, dummy).
- `LIMIT_NUMBER_OF_TRUCKS_HELPER`: Максимальное количество грузовиков для использования.
- `TRUCK_SIZE_HELPER`: Размер грузовика в формате HEIGHTxWIDTH (например, 6x6).
- `BOX_NAMES_HELPER`: Названия уже сохраненных коробок, которые нужно упаковать, корректный ввод через `","` без пробелов. Пример: `Штанга,Кот в коробке`.
# README: BoxController

## Описание

`BoxController` предоставляет интерфейс для управления коробками в системе. Он позволяет создавать, получать, обновлять и удалять коробки через командную оболочку Spring Shell.

## Входные Данные

### Команды и Их Параметры

Команды доступны через интерфейс командной строки. Вот параметры, которые можно использовать с командами управления коробками:

1. **Получить все коробки**
    - Команда: `get-all-boxes`
    - Описание: Получить список всех коробок.

   ```shell
   get-all-boxes
   ```

2. **Получить коробку по имени**
    - Команда: `get-box-by-name`
    - Параметры:
        - `--boxName`: Имя коробки, которую нужно получить (обязательный).
    - Описание: Получить информацию о конкретной коробке по её имени.

   ```shell
   get-box-by-name --boxName имя_коробки
   ```

3. **Создать новую коробку**
    - Команда: `create-box`
    - Параметры:
        - `--boxName`: Имя создаваемой коробки (обязательный).
        - `--form`: Форма коробки (обязательный).
        - `--symbol`: Символ, представляющий коробку (обязательный).
    - Описание: Создать новую коробку с заданными параметрами.

   ```shell
   create-box --boxName имя_коробки --form форма_коробки --symbol символ
   ```

4. **Обновить коробку по имени**
    - Команда: `update-box-by-name`
    - Параметры:
        - `--boxName`: Имя коробки для обновления (обязательный).
        - `--form`: Новая форма коробки (обязательный).
        - `--symbol`: Новый символ для коробки (обязательный).
    - Описание: Обновить информацию о конкретной коробке.

   ```shell
   update-box-by-name --boxName имя_коробки --form новая_форма --symbol новый_символ
   ```

5. **Удалить коробку по имени**
    - Команда: `delete-box-by-name`
    - Параметры:
        - `--boxName`: Имя коробки, которую нужно удалить (обязательный).
    - Описание: Удалить коробку из системы.

   ```shell
   delete-box-by-name --boxName имя_коробки
   ```

## Ограничения

1. **Имена коробок**: Имена должны быть уникальными в системе и не должны содержать специальных символов.
2. **Форма и символ**: Убедитесь, что форма и символ соответствуют ожиданиям системы. Символ должен состоять из одного символа.

## Заключение

Используйте приведенные команды для управления коробками в системе. Убедитесь в правильности введенных параметров, чтобы избежать ошибок при выполнении операций.
